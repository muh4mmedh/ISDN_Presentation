Problem Statement

Small-to-medium enterprises in Sri Lanka rely on manual spreadsheets and disconnected legacy tools for inventory tracking, invoicing, and financial reporting. This leads to stock discrepancies, delayed procurement, revenue leakage, and poor decision-making due to lack of real-time data visibility.

  • Automate end-to-end inventory lifecycle
  • Real-time multi-location stock tracking
  • Streamline procurement to payment cycle
  • Customer self-service portal for orders
  • Role-based access with audit trails

üí° Solution

  • Web-based 3-tier architecture
  • Database-centric business logic (SPs)
  • Real-time WebSocket notifications
  • Dual authentication (Staff + Customer)
  • Comprehensive reporting dashboard

Customer Portal ‚Äî New Feature

A self-service portal enabling registered debtors/customers to browse inventory, place orders online, and view their invoice history ‚Äî extending the system beyond internal staff use.

Authentication Flow

  • Dual login: Staff (username/location) vs Customer (Gmail)
  • Customer registers using debtor code from sales rep
  • bcrypt password hashing (separate from staff VB6 hash)
  • Dedicated JWT cookie (customer_jwt)
  • cDebtorCode as unique customer identifier (PK)

Portal Features

  • Product catalog with search & category filter
  • Shopping cart with quantity management
  • Order placement (auto-creates invoice)
  • Order history with status tracking
  • Customer profile with credit limit info

Security Implementation

Authentication

  • Staff: VB6-compatible XOR encryption (legacy)
  • Customer: bcrypt hashing (modern)
  • JWT tokens (HS256) with 24-hour expiry
  • HttpOnly cookies (SameSite=None)
  • Separate sessions: jwt, cashier_jwt, customer_jwt

Authorization (RBAC)

  • tblSecurityRights: Module ‚Üí Screen ‚Üí Event hierarchy
  • Rights assigned to users OR security groups via tblSecurityGroupRelation
  • INVENTORYWEB module for all web permissions
  • tblUserLocation for location-based restrictions
  • 3-tier middleware chain in Fiber (AuthRequired, CashierAuth, CustomerAuth)

Data Protection

CORS Whitelist SQL Injection Prevention (Parameterized SPs) HTTPS Ready Input Validation XSS Protection
